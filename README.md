# Лабораторна робота №4
## Дослідження роботи з динамічною пам’яттю у Linux

# Завдання 4.1
## Максимальний обсяг пам’яті для malloc()

Було створено програму, яка намагається виділити дуже великий обсяг пам’яті за допомогою функції malloc().

Було встановлено, що аргумент функції malloc має тип size_t.  
На 64-бітній системі розмір типу size_t становить 8 байтів.

Теоретичний максимальний обсяг пам’яті: 2^64 байт

Однак у Linux процесу доступна лише половина адресного простору, тому максимальний обсяг становить приблизно: 2^63 байт ≈ 8 EB.

# Завдання 4.2
## Поведінка malloc при від’ємному аргументі

Було досліджено ситуацію, коли у malloc передається значення, отримане в результаті арифметичного виразу з від’ємним числом.

Було встановлено, що через переповнення цілого типу (integer overflow) може утворитися дуже велике додатне число, яке передається у malloc. У такому випадку функція може намагатися виділити дуже великий блок пам’яті або повернути NULL.

# Завдання 4.3
## Поведінка malloc(0)

Було перевірено поведінку функції malloc при виклику з аргументом 0.

Результати показали, що malloc(0) може:

- повернути NULL
- повернути унікальний вказівник

Такий вказівник дозволено передавати у функцію free().

# Завдання 4.4
## Аналіз помилки у коді

Було проаналізовано код, у якому пам’ять звільняється за допомогою free(), але вказівник після цього не встановлюється у NULL.

Це призводить до появи так званого "висячого вказівника" (dangling pointer).  
У наступних ітераціях програма може повторно звільняти вже звільнену пам’ять, що викликає помилку double free.

Правильним рішенням є встановлення вказівника у NULL після виклику free().


# Завдання 4.5
## Поведінка realloc при помилці виділення пам’яті

Було досліджено ситуацію, коли realloc не може виділити необхідний обсяг пам’яті.

У такому випадку функція realloc повертає NULL, але початковий блок пам’яті залишається дійсним.  
Тому програміст повинен самостійно звільнити стару пам’ять.


# Завдання 4.6
## Особливі випадки realloc()

Було перевірено два спеціальні випадки роботи realloc:

realloc(NULL, size) — працює аналогічно malloc(size)

realloc(ptr, 0) — еквівалентно виклику free(ptr)


# Завдання 4.7
## Використання reallocarray()

Було переписано програму з використанням функції reallocarray().

Ця функція є безпечнішою, оскільки перевіряє переповнення при множенні значень n * size перед виділенням пам’яті.


# Завдання 3
## Дослідження продуктивності malloc/free у потоках

Було досліджено продуктивність операцій malloc/free при використанні різної кількості потоків.

Тестування проводилося для:

- 1 потоку
- 2 потоків
- 4 потоків
- 8 потоків

Було також реалізовано оптимізований варіант із використанням thread-local memory pool.

Результати показали, що використання локального пулу пам’яті зменшує конкуренцію між потоками та підвищує продуктивність програми.


# Висновок

У ході лабораторної роботи було досліджено особливості роботи функцій malloc(), free(), realloc() та reallocarray() у Linux.

Було вивчено типові помилки роботи з динамічною пам’яттю, такі як dangling pointer та double free. Також було досліджено продуктивність операцій виділення пам’яті у багатопотоковому середовищі.

Отримані результати показують важливість правильного управління пам’яттю для забезпечення стабільної та ефективної роботи програм.
